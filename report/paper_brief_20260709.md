### 1. Beware of Agentic Botnets: Scalable Untargeted Promptware Attacks via Universal and Transferable Adversarial HalluSquatting

- **arXiv ID**: 2607.07433v1 · **원문**: https://arxiv.org/abs/2607.07433v1
- **저자**: Aya Spira, Stav Cohen, Elad Feldman, Ron Bitton, Avishai Wool, Ben Nassi
- **게재일**: 2026-07-08 · **카테고리**: cs.CR
- **요약 근거**: 본문 기반
- ⚠️ [인젝션 의심] 본문에 지시문 형태의 문구가 감지됨 — 요약 내용을 사람이 검토할 것

**핵심 기여**

- 저자들은 LLM 애플리케이션에서의 새로운 위협인 프롬프트웨어(promptware)를 소개하며, 이를 통해 공격자가 직접적인 채널 없이도 대규모로 공격을 수행할 수 있음을 보여준다.
- 아드버서리 할루스쿼팅(adversarial HalluSquatting)이라는 기법을 통해 공격자가 인기 있는 리소스를 사전 등록하고, 이를 통해 LLM 애플리케이션을 악용하여 봇넷을 형성할 수 있는 방법을 제시한다.
- 이 연구는 다양한 LLM 애플리케이션에서의 공격 가능성을 실증적으로 보여주며, 원격 코드 실행 및 도구 호출을 통해 공격의 효과를 입증한다.

**방법론**

저자들은 LLM 애플리케이션이 외부 리소스를 검색하는 방식을 분석하고, LLM의 할루시네이션 경향을 이용하여 공격자가 리소스 식별자를 조작할 수 있는 방법을 제시한다. 이를 통해 공격자는 인기 있는 리소스를 사전 등록하고, 사용자가 이를 요청할 때 악성 코드를 포함한 리소스를 반환하도록 유도한다. 실험을 통해 다양한 LLM 애플리케이션에서의 공격 성공률을 측정하고, 이를 통해 공격의 효과성을 입증한다.

**실무 적용 포인트** *(에이전트 의견 — 논문 내용과 구분됨)*

- 실무자들은 LLM 애플리케이션의 외부 리소스 검색 기능을 점검하고, 할루시네이션으로 인한 보안 취약점을 인식해야 한다.
- 프롬프트웨어 공격을 방지하기 위해, LLM 애플리케이션의 입력 검증 및 리소스 검색 메커니즘을 강화할 필요가 있다.
- 공격자가 사전 등록한 리소스에 대한 모니터링 및 경고 시스템을 구축하여, 비정상적인 리소스 요청을 탐지할 수 있어야 한다.

**한계**

- 연구는 특정 LLM 애플리케이션에 대한 공격을 중심으로 진행되었으며, 모든 LLM 애플리케이션에 대한 일반화는 제한적일 수 있다.
- 실험에서 사용된 데이터셋과 환경이 특정 조건에 국한되어 있어, 다양한 환경에서의 공격 가능성에 대한 추가 연구가 필요하다.

### 2. SA-DRL: Security-Aware Deep Reinforcement Learning for Ransomware Detection with Asymmetric Reward Design

- **arXiv ID**: 2607.06880v1 · **원문**: https://arxiv.org/abs/2607.06880v1
- **저자**: Jannatul Ferdous, Rafiqul Islam, Md Zahidul Islam
- **게재일**: 2026-07-08 · **카테고리**: cs.CR
- **요약 근거**: 본문 기반

**핵심 기여**

- SA-DRL 프레임워크는 비대칭 오차 비용을 보상 신호에 통합하여 랜섬웨어 탐지에서 놓친 탐지를 줄이는 데 중점을 둡니다.
- SOMS 기준을 도입하여 최소한의 거짓 부정률, 최대 F1 점수 및 최소 훈련 시간을 우선시하는 보안 최적 모델 선택 방법을 제안합니다.
- 네 가지 심층 강화 학습 알고리즘(DQN, DDQN, PPO, A2C)을 비교하여 비대칭 보상 설계의 효과를 실험적으로 검증합니다.

**방법론**

SA-DRL 프레임워크는 랜섬웨어 탐지를 위한 마르코프 결정 프로세스(MDP)를 기반으로 하며, 비대칭 보상 설계를 통해 놓친 탐지를 최소화하도록 최적화됩니다. 이 프레임워크는 샘플 순서를 적응적으로 조정하고, 보상 신호에 비대칭 비용을 통합하여 강화 학습 에이전트가 보안 우선 결정을 내리도록 합니다. 실험은 480회의 훈련을 통해 수행되었습니다.

**실무 적용 포인트** *(에이전트 의견 — 논문 내용과 구분됨)*

- 랜섬웨어 탐지 시스템에 비대칭 보상 설계를 적용하여 보안 민감한 환경에서의 탐지 성능을 향상시킬 수 있습니다.
- 보안 최적 모델 선택(SOMS) 기준을 통해 실무에서 모델 선택 시 보안 우선 순위를 반영할 수 있습니다.
- 비대칭 보상 설계는 랜섬웨어 탐지에서 놓친 탐지를 줄이는 데 효과적이므로, 실무자들은 이를 적용하여 탐지 시스템의 신뢰성을 높일 수 있습니다.

**한계**

- 제안된 방법론은 특정 랜섬웨어 변종에 대한 성능을 기반으로 하므로, 다양한 변종에 대한 일반화 가능성에 한계가 있을 수 있습니다.
- 비대칭 보상 설계의 효과는 실험 환경에 따라 다를 수 있으며, 실제 운영 환경에서의 성능은 추가적인 검증이 필요합니다.

### 3. Auditable Machine Unlearning for Privacy-Compliant Ransomware Detection Using Multi-Shard SISA and Deep Reinforcement Learning

- **arXiv ID**: 2607.06860v1 · **원문**: https://arxiv.org/abs/2607.06860v1
- **저자**: Jannatul Ferdous, Rafiqul Islam, Md Zahidul Islam
- **게재일**: 2026-07-07 · **카테고리**: cs.CR
- **요약 근거**: 본문 기반

**핵심 기여**

- 저자들은 DDQN과 다중 샤드 SISA를 통합한 프레임워크를 제안하여, 랜섬웨어 탐지 및 데이터 삭제의 효율성을 높였다고 주장한다.
- 이 프레임워크는 삭제 요청 시 선택적 샤드 수준 재훈련을 통해 개인정보 보호 규정을 준수하면서도 높은 탐지 성능을 유지할 수 있음을 보여준다.
- 또한, 저자들은 오라클 기반의 삭제 검증 및 멤버십 추론 감사 메커니즘을 도입하여, 삭제된 샘플이 모델에 미치는 영향을 효과적으로 평가할 수 있다고 보고한다.

**방법론**

이 연구는 랜섬웨어 탐지를 위한 DDQN 기반의 강화 학습 모델과 다중 샤드 SISA 훈련을 결합한 프레임워크를 제안한다. 이 프레임워크는 삭제 요청이 발생할 경우, 영향을 받는 샤드만 재훈련하여 전체 모델을 재구성하는 데 드는 비용을 줄인다. 실험은 유틸리티 보존, 오라클 기반의 삭제 검증, 멤버십 추론 감사 및 계산 효율성을 평가하는 방식으로 진행되었다.

**실무 적용 포인트** *(에이전트 의견 — 논문 내용과 구분됨)*

- 랜섬웨어 탐지 시스템에서 개인정보 보호 규정을 준수하기 위해, 선택적 데이터 삭제 기능을 구현하는 것이 중요하다.
- 다중 샤드 SISA 접근 방식을 통해, 실시간으로 데이터 삭제 요청에 대응할 수 있는 효율적인 시스템 설계가 가능하다.
- 강화 학습 기반의 탐지 모델은 동적인 랜섬웨어 행동에 적응할 수 있는 유연성을 제공하므로, 지속적인 업데이트가 필요하다.

**한계**

- 다중 샤드 설정에서 높은 삭제 비율과 반복적인 삭제 요청이 있을 경우, 모델 성능이 급격히 저하될 수 있다.
- 제안된 프레임워크는 특정 환경에서만 평가되었으므로, 다양한 실제 환경에서의 성능 검증이 필요하다.

### 4. ProvICS: A Provenance-based Intrusion Detection for Industrial Control Systems

- **arXiv ID**: 2607.05989v1 · **원문**: https://arxiv.org/abs/2607.05989v1
- **저자**: Md Neyamul Islam Shibbir, Deepak K Tosh
- **게재일**: 2026-07-07 · **카테고리**: cs.CR
- **요약 근거**: 본문 기반

**핵심 기여**

- 저자들은 산업 제어 시스템(ICS)에서의 침입 탐지를 위한 새로운 오픈 소스 데이터셋인 ProvICS를 제안한다.
- 이 데이터셋은 전체 시스템의 원천 추적 그래프, Modbus 패킷 검사 기록, 물리적 프로세스 텔레메트리 등 네 가지 스트림을 동기화하여 수집한다.
- ProvICS는 48시간의 정상 운영과 22시간의 공격 데이터를 포함하며, 32개의 공격 이벤트를 통해 다양한 ICS ATT&CK 기술을 다룬다.

**방법론**

이 연구에서는 하드웨어-인-더-루프(HIL) CPS 테스트베드를 설계하여 산업 화학 반응기 제어 아키텍처를 재현하고, 이를 통해 ProvICS라는 다중 모달 데이터셋을 수집하였다. 데이터셋은 호스트 수준의 원천 추적, 산업 네트워크 트래픽, 물리적 프로세스 상태를 포함하여, 각 모달리티의 데이터를 시간 동기화하여 분석할 수 있도록 구성되었다.

**실무 적용 포인트** *(에이전트 의견 — 논문 내용과 구분됨)*

- ProvICS 데이터셋은 산업 제어 시스템의 침입 탐지 연구에 있어 중요한 기초 자료로 활용될 수 있다.
- 다중 모달 데이터의 융합을 통해 공격 탐지의 정확성을 높일 수 있는 가능성을 제시한다.
- 실제 ICS 환경에서의 침입 탐지 시스템 개발에 있어, 원천 추적 기반 접근 방식의 유용성을 강조한다.

**한계**

- 현재 테스트베드는 단일 PLC와 디지털 트윈 환경을 기반으로 하여 대규모 환경을 대표하지 않는다.
- 실제 물리적 플랜트 통합이 이루어지지 않아, 실제 환경에서의 적용 가능성에 한계가 있다.

### 5. Hidden Amplifiers: Cross-Level Risk in Software Supply Chains

- **arXiv ID**: 2607.05894v1 · **원문**: https://arxiv.org/abs/2607.05894v1
- **저자**: Rakesh Podder, Rafael Fabian Gonzalez Arellano, Indrajit Ray
- **게재일**: 2026-07-07 · **카테고리**: cs.SE, cs.CR
- **요약 근거**: 본문 기반

**핵심 기여**

- 저자들은 코드 수준의 위험 지표와 생태계 수준의 의존성 노출을 통합하는 '교차 수준 위험 전파' 프레임워크를 제안한다.

**방법론**

### 6. Untrusted Content Masking for Web Agents with Security Guarantees

- **arXiv ID**: 2607.05277v1 · **원문**: https://arxiv.org/abs/2607.05277v1
- **저자**: Kristina Nikolić, Egor Zverev, Javier Rando, Matthew Jagielski, Edoardo Debenedetti, Florian Tramèr
- **게재일**: 2026-07-06 · **카테고리**: cs.CR, cs.LG
- **요약 근거**: 본문 기반
- ⚠️ [인젝션 의심] 본문에 지시문 형태의 문구가 감지됨 — 요약 내용을 사람이 검토할 것

**핵심 기여**

- 저자들은 Untrusted Content Masking (UCM)이라는 방어 기법을 제안하여 웹 환경에서 신뢰할 수 없는 콘텐츠와 신뢰할 수 있는 콘텐츠 간의 경계를 복원한다고 주장한다.
- UCM은 웹 페이지의 Document Object Model (DOM)을 활용하여 신뢰할 수 없는 영역을 식별하고, 이를 마스킹하여 에이전트가 악의적인 콘텐츠에 노출되지 않도록 한다.
- 이 연구는 UCM이 다양한 웹사이트에서 작업 유용성을 유지하면서도 보안 보장을 제공함을 실증적으로 검증했다고 보고한다.

**방법론**

이 논문에서는 Untrusted Content Masking (UCM)이라는 방어 기법을 제안한다. UCM은 웹 페이지의 DOM 구조를 분석하여 신뢰할 수 없는 콘텐츠를 마스킹하고, 에이전트가 신뢰할 수 있는 콘텐츠와 상호작용할 수 있도록 한다. 또한, 신뢰할 수 없는 콘텐츠에 접근해야 할 경우, 에이전트는 Quarantined Model (Q-Model)을 통해 제한된 형태의 응답을 받아 안전하게 작업을 수행할 수 있다.

**실무 적용 포인트** *(에이전트 의견 — 논문 내용과 구분됨)*

- 웹 에이전트를 개발하는 실무자들은 UCM을 통해 웹 환경에서의 보안을 강화할 수 있다.
- 신뢰할 수 없는 콘텐츠를 효과적으로 관리하기 위해 웹사이트 소유자들이 콘텐츠에 신뢰 경계를 명시하는 것이 중요하다.
- UCM의 구현은 기존 웹사이트에 쉽게 통합될 수 있으며, 보안성을 높이면서도 사용자 경험을 저해하지 않는다.

**한계**

- UCM은 신뢰할 수 없는 콘텐츠의 경계를 정확히 식별하기 위해 웹사이트 소유자가 적절한 레이블을 제공해야 한다는 전제가 있다.
- 자동 경계 식별 방법이 모든 웹사이트에 대해 항상 정확하게 작동하지 않을 수 있다.

### 7. Agent Data Injection Attacks are Realistic Threats to AI Agents

- **arXiv ID**: 2607.05120v1 · **원문**: https://arxiv.org/abs/2607.05120v1
- **저자**: Woohyuk Choi, Juhee Kim, Taehyun Kang, Jihyeon Jeong, Luyi Xing, Byoungyoung Lee

**핵심 기여**

- 저자들은 새로운 공격 유형인 에이전트 데이터 주입 공격(ADI)을 소개하며, 이는 신뢰할 수 있는 데이터로 위장한 악성 데이터를 주입하여 AI 에이전트가 의도하지 않은 행동을 하도록 유도한다고 주장한다.
- 이 논문은 ADI가 기존의 방어 메커니즘을 우회할 수 있는 여러 실제 사례를 제시하며, 웹 에이전트와 코딩 에이전트에서의 취약점을 평가한다.

**방법론**

이 연구는 ADI 공격의 개념을 정의하고, 이를 실현하기 위한 핵심 기술인 확률적 구분자 주입을 설명한다. 저자들은 여러 실제 AI 에이전트를 대상으로 ADI 공격을 시연하고, 공격의 성공률을 평가하여 기존 방어 메커니즘의 한계를 분석한다.

**실무 적용 포인트** *(에이전트 의견 — 논문 내용과 구분됨)*

- 기존의 방어 메커니즘이 ADI 공격에 취약하므로, 새로운 방어 전략을 개발할 필요가 있다.
- 실제 환경에서 AI 에이전트를 사용할 때, 외부 데이터의 출처를 검증하는 절차를 강화해야 한다.

**한계**

- 논문에서 제시된 ADI 공격은 특정 AI 에이전트에 대한 사례 연구에 국한되어 있으며, 모든 AI 시스템에 일반화할 수 있는지는 불확실하다.
- 실험에서 사용된 모델과 환경이 제한적이어서, 다양한 환경에서의 ADI 공격의 효과는 추가 연구가 필요하다.

### 10. DualView: Preventing Indirect Prompt Injection in Personal AI Agents

- **arXiv ID**: 2607.03821v1 · **원문**: https://arxiv.org/abs/2607.03821v1
- **저자**: Juhee Kim, Woohyuk Choi, Taehyun Kang, Youngmin Kim, Byoungyoung Lee
- **게재일**: 2026-07-04 · **카테고리**: cs.CR, cs.AI
- **요약 근거**: 본문 기반
- ⚠️ [인젝션 의심] 본문에 지시문 형태의 문구가 감지됨 — 요약 내용을 사람이 검토할 것

**핵심 기여**

- DualView는 개인 AI 에이전트의 보안을 강화하기 위해 신뢰할 수 없는 데이터를 사용자 환경에서도 추적할 수 있도록 확장된 방어 메커니즘을 제안한다.
- AgentView와 HumanView라는 두 가지 시각을 제공하여, 에이전트는 신뢰할 수 없는 데이터를 기호로만 인식하고, 사용자는 원본 데이터를 그대로 사용할 수 있게 한다.
- DualView는 기존의 공격 템플릿에 국한되지 않고, 모든 IPI 공격을 차단할 수 있는 설계를 통해 보안성을 높인다.

**방법론**

DualView는 에이전트의 컨텍스트와 사용자 환경 모두에서 신뢰할 수 없는 데이터를 추적하고 격리하는 방식을 채택한다. 에이전트는 AgentView를 통해 신뢰할 수 없는 데이터를 기호로 인식하고, HumanView를 통해 사용자는 원본 데이터를 사용할 수 있도록 한다. 이 시스템은 OpenClaw 플러그인으로 구현되어 기존 도구 호출 로직이나 도구 구현을 변경하지 않고도 배포할 수 있다.

**실무 적용 포인트** *(에이전트 의견 — 논문 내용과 구분됨)*

- 개인 AI 에이전트를 사용하는 기업에서는 DualView와 같은 보안 메커니즘을 도입하여 내부 데이터 유출을 방지할 수 있다.
- 신뢰할 수 없는 데이터를 안전하게 처리하기 위해, 사용자와 에이전트 간의 데이터 동기화 및 관리 정책을 수립하는 것이 중요하다.
- AI 에이전트의 보안성을 높이기 위해, 사용자 교육과 함께 DualView와 같은 기술적 방어 수단을 병행해야 한다.

**한계**

- DualView는 사용자가 자신의 시스템을 공격하는 경우에는 보호하지 않으며, 이는 사용자 신뢰 모델에 기반한 한계이다.
- 기존의 명령어 실행 패턴을 완벽하게 탐지하지 못할 수 있으며, 이는 추가적인 모니터링이 필요함을 의미한다.
